# 修复正则查询查看全部的高亮显示问题

## 问题描述

在正则查询的"查看全部"功能中，虽然内容已经根据正则高亮了，但是：
1. 初始显示时没有正确的高亮
2. 切换格式（原始文本/智能格式/分段显示）时高亮丢失

## 问题原因

在 `viewFileWithRegex` 函数中，直接将格式化后的内容插入到 `modal-content` 中，没有使用 `content-display` 容器：

```javascript
// ❌ 错误：直接显示内容，没有 content-display 容器
document.getElementById('modal-content').innerHTML = toolbar + smartFormatted;
```

而 `setFormat` 函数是通过操作 `content-display` 元素来切换格式的：

```javascript
function setFormat(format) {
    const display = document.getElementById('content-display');
    // ...
    if (format === 'raw') {
        display.innerHTML = highlightRegexContent(content, pattern);
    }
    // ...
}
```

因为没有 `content-display` 元素，所以切换格式时找不到目标元素，导致高亮失效。

## 解决方案

修改 `viewFileWithRegex` 函数，将内容包裹在 `content-display` 容器中：

```javascript
// ✅ 正确：创建 content-display 容器
document.getElementById('modal-content').innerHTML = toolbar + 
    '<div id="content-display" class="formatted-content">' + smartFormatted + '</div>';
```

## 修改内容

### viewFileWithRegex 函数

**修改前**：

```javascript
    smart: smartFormatted,
```

**修改后**：


## 高亮逻辑

### formatSmartContent 函数
```javascript
function formatSmartContent(content, keywords, exactMatch = false, pattern = '') {
    let highlightFunc;
    if (pattern) {
        // 使用正则高亮所有匹配的内容
        highlightFunc = (text, kw) => highlightRegexContent(text, pattern);
    } else {
        // 使用关键词高亮
        highlightFunc = exactMatch ? highlightAndEscapeExact : highlightAndEscape;
    }
    // ... 格式化逻辑
}
```

### highlightRegexContent 函数
```javascript
function highlightRegexContent(text, pattern) {
    const escapedText = escapeHtml(text);
    const regex = new RegExp(pattern, 'gi');
    return escapedText.replace(regex, "<span class='highlight'>$&</span>");
}
```
