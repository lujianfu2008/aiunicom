**错误 2**：bytes 与 str 混用
```
2026-03-24 00:48:46,057 - ERROR - 正则搜索失败：startswith first arg must be str or a tuple of str, not bytes
```

**原因**：在 `vector_store.py` 的 `find_by_regex()` 方法中，`startswith()` 使用了 bytes 参数

**问题代码**：
```python
valid_keys = [
    k for k in doc_keys 
    if not k.decode().endswith('_idx') 
    and not k.decode().endswith('_cache')
    and not k.decode().startswith(b'workorder_fp:')  # ❌ 错误：使用 bytes
    and not k.decode().endswith('stats_cache')
]
```

**修复**：
```python
valid_keys = [
    k for k in doc_keys 
    if not k.decode().endswith('_idx') 
    and not k.decode().endswith('_cache')
    and not k.decode().startswith('workorder_fp:')  # ✅ 正确：使用 str
    and not k.decode().endswith('stats_cache')
]
```

## 经验教训

1. **类型导入完整性**
   - 使用类型提示时，确保导入所有需要的类型
   - 常用类型：`Dict`, `Any`, `List`, `Optional`, `Tuple`, `Union`

2. **bytes 与 str 的处理**
   - Redis 返回的键是 bytes 类型
   - 使用 `.decode()` 后转换为 str
   - 后续操作必须使用 str 类型

3. **测试覆盖**
   - 单元测试很重要
   - 集成测试能发现更多问题
   - 实时日志监控有助于快速定位问题
