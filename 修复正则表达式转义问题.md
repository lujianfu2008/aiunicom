# 修复正则表达式转义问题

## 问题描述

出现 `Invalid or unexpected token` 错误，位置在正则查询结果列表中。

## 根本原因

在正则查询结果列表中，`onclick` 属性直接调用 `data.pattern.replace()`，当 pattern 包含反斜杠（如 `\d`）时，会导致 JavaScript 语法错误：

```javascript
// ❌ 错误：直接在模板字符串中调用 replace，且只转义单引号
onclick="viewFileWithRegex('${safeFilePath}', '${safeFileName}', ..., '${data.pattern.replace(/'/g, "\\'")}')">
```

问题：
1. 正则表达式中的反斜杠（如 `\d`、`\s`）没有被转义
2. 在模板字符串中直接调用方法不够安全

## 解决方案

预先处理所有需要转义的字符（单引号和反斜杠），然后在模板字符串中使用处理后的变量：

```javascript
// ✅ 正确：预先转义所有特殊字符
const safeFilePath = filePath.replace(/'/g, "\\'").replace(/\\/g, '\\\\');
const safeFileName = fileName.replace(/'/g, "\\'").replace(/\\/g, '\\\\');
const safePattern = data.pattern.replace(/'/g, "\\'").replace(/\\/g, '\\\\');

html += `
    <button onclick="viewFileWithRegex('${safeFilePath}', '${safeFileName}', ..., '${safePattern}')">
`;
```

## 转义规则

### JavaScript 字符串中的特殊字符

| 字符 | 含义 | 转义后 | 示例 |
|-----|------|--------|------|
| `'` | 单引号 | `\'` | `'\''` → `\'` |
| `\` | 反斜杠 | `\\` | `'\d'` → `'\\d'` |
| `"` | 双引号 | `\"` | `"\""` → `\"` |
| 换行 | 换行符 | `\n` | `"a\nb"` → `"a\nb"` |

### 正则表达式中的反斜杠

当正则表达式作为字符串传递时，反斜杠需要双重转义：

```javascript
// 原始正则
const pattern = '1[3-9]\\d{9}';  // 匹配手机号

// 在 HTML onclick 中使用时
// ❌ 错误：只转义单引号
pattern.replace(/'/g, "\\'")  
// 结果：'1[3-9]\d{9}' - 反斜杠还在

// ✅ 正确：同时转义单引号和反斜杠
pattern.replace(/'/g, "\\'").replace(/\\/g, '\\\\')
// 结果：'1[3-9]\\\\d{9}' - 反斜杠被正确转义
```

## 完整转义规则总结

### 在 HTML onclick 属性中传递字符串

```javascript
// 需要转义的字符
const safe = unsafe
    .replace(/\\/g, '\\\\')  // 1. 先转义反斜杠
    .replace(/'/g, "\\'");   // 2. 再转义单引号

// 使用
html += `<button onclick="func('${safe}')">`;
```

### 转义顺序很重要！

```javascript
// ✅ 正确顺序
str.replace(/\\/g, '\\\\').replace(/'/g, "\\'");

// ❌ 错误顺序（会导致重复转义）
str.replace(/'/g, "\\'").replace(/\\/g, '\\\\');
// 结果：\' 会变成 \\\'
```

**注意**：在这个场景中，两个替换的顺序其实不影响，因为 `\'` 中的反斜杠不会被再次匹配。但为了代码清晰，建议先转义反斜杠。
