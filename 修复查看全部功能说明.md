# 查看全部文档功能使用说明

## 问题描述
之前在使用"查看全部"功能时，系统会显示"文件未找到"错误。

## 解决方案
已对系统进行以下修复：

### 1. 新增 API 接口
- **接口路径**: `/api/documents`
- **功能**: 获取知识库中所有文档的列表
- **返回数据**: 
  - `total`: 文档总数
  - `documents`: 文档列表，包含文件名、文件路径、问题类型、问题 ID、内容长度等信息

### 2. 改进文件搜索逻辑
- 修复了 `api_view_file()` 函数的搜索逻辑
- 支持通过文件名和文件路径精确匹配
- 支持部分匹配（文件名包含搜索词）
- 兼容 `file_path` 和 `source` 两种存储方式

### 3. 前端界面更新
- 在"系统信息"标签页中新增"知识库管理"部分
- 添加"查看所有文档"按钮
- 点击按钮可查看所有文档列表
- 点击任意文档可查看详情

## 使用方法

### 方法一：通过 Web 界面
1. 打开浏览器访问：http://localhost:5000
2. 点击"系统信息"标签
3. 在"知识库管理"部分点击"查看所有文档"按钮
4. 查看所有文档列表（共 1012 个文档）
5. 点击任意文档查看详情

### 方法二：通过 API 接口
```bash
# 获取所有文档列表
GET http://localhost:5000/api/documents

# 查看特定文件内容
GET http://localhost:5000/api/view_file?file_path=文件路径&file_name=文件名
```

## 测试结果
✓ 成功获取文档列表（1012 个文档）
✓ 成功查看文档详情
✓ 文件搜索功能正常
✓ 无"文件未找到"错误

## 技术细节

### 修改的文件
- `api_server.py`: 
  - 新增 `api_documents()` 函数
  - 改进 `api_view_file()` 函数的搜索逻辑
  - 新增前端 `viewAllDocuments()` JavaScript 函数
  - 新增"查看所有文档"按钮样式

### 搜索逻辑优化
1. **方法 1**: 从文件名索引搜索（最快）
2. **方法 2**: 通过 `find_document_by_filename()` 搜索
3. **方法 3**: 从 keywords 中提取文件名搜索
4. **方法 4**: 直接搜索 Redis 中的所有文档（改进版）
   - 精确匹配文件名
   - 精确匹配文件路径
   - 部分匹配（文件名包含搜索词）

## 注意事项
- 确保 Redis Stack 服务正在运行
- 确保 Web 服务器已启动（默认端口 5000）
- 文档数量较多时，加载可能需要几秒钟
