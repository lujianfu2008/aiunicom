# 彻底修复字符串转义问题

## 问题描述

再次出现 `Invalid or unexpected token` 错误，导致整个 JavaScript 无法加载。

## 根本原因

在 `formatSectionContent` 函数中，还有两处使用了错误的转义：

```javascript
// ❌ 错误：双反斜杠
let lines = section.trim().split('\\n');
result += ...replace(/\\n/g, '<br>')...
```

## JavaScript 转义详解

### 在普通字符串中

```javascript
// 普通字符串
'\n'   // ✅ 换行符（1 个字符）
'\\n'  // ❌ 反斜杠 + n（2 个字符）

// 拆分字符串
str.split('\n')    // ✅ 按换行符拆分
str.split('\\n')   // ❌ 按字面意义的 "\n" 拆分
```

### 在正则表达式中

```javascript
// 正则表达式
/\n/    // ✅ 匹配换行符
/\\n/   // ❌ 匹配反斜杠 + n

// 替换
str.replace(/\n/g, '<br>')    // ✅ 替换所有换行符为<br>
str.replace(/\\n/g, '<br>')   // ❌ 替换所有"\n"字符串为<br>
```

### 在 Python 字符串中的 JavaScript 代码

当在 Python 字符串中嵌入 JavaScript 代码时，需要特别注意：

```python
# Python 字符串中的 JavaScript
js_code = """
    // Python 会先解释一遍转义
    let lines = content.split('\\n');  // Python 解释后变成 \n，JavaScript 收到的是 \\n
    let lines = content.split('\n');   // Python 解释后变成 换行符，JavaScript 收到的是 \n
"""
```

**规则**：
- 在 Python 的 `"""` 字符串中，`\\n` 会被 Python 解释为 `\n`，然后 JavaScript 收到的是字面意义的 `\n`（两个字符）
- 在 Python 的 `"""` 字符串中，`\n` 会被 Python 解释为换行符，JavaScript 收到的就是换行符

## 修复内容

### 1. formatSectionContent - 行拆分

**修改前**：
```javascript

```

**修改后**：
```javascript
let lines = section.trim().split('\n');
```

### 2. formatSectionContent - 换行符替换

**修改前**：


## 经验总结

### Python 字符串中的转义规则

当在 Python 字符串中编写 JavaScript 代码时：

1. **普通字符串**：使用单反斜杠
   ```python
   js = "str.split('\n')"  # ✅ JavaScript 收到：str.split('\n')
   ```

2. **正则表达式**：使用单反斜杠
   ```python
   js = "str.replace(/\n/g, '<br>')"  # ✅ JavaScript 收到：str.replace(/\n/g, '<br>')
   ```

3. **如果要表示字面意义的反斜杠**：使用双反斜杠
   ```python
   js = "str.split('\\\\n')"  # ✅ JavaScript 收到：str.split('\\n')
   ```

2. **查看源代码**：在浏览器中右键 → 查看源代码，确认 JavaScript 代码正确
3. **使用 console.log**：在关键位置添加日志，查看变量值
