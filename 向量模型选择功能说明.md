# 向量模型选择功能说明

## 功能概述

系统现在支持在配置界面选择使用哪个向量模型进行文本向量化：

1. **本地模型 (text2vec)** - 使用本地部署的 text2vec-base-chinese 模型
2. **智谱 AI (embedding-2)** - 使用智谱 AI 的在线向量模型

## 配置位置

在系统信息页面的"配置信息"选项卡中，新增了"向量模型配置"部分。

### 访问路径

1. 打开系统首页
2. 点击顶部导航栏的"系统信息"
3. 滚动到"配置信息"部分
4. 找到"向量模型提供商"下拉框

## 模型对比

| 特性 | 本地模型 | 智谱 AI |
|-----|---------|--------|
| **速度** | 快（本地计算） | 较慢（网络请求） |
| **精度** | 良好 | 优秀 |
| **网络依赖** | 无需网络 | 需要网络连接 |
| **资源占用** | 占用内存和 CPU | 无本地资源占用 |
| **适用场景** | 频繁搜索、离线环境 | 追求更高精度 |

## 配置说明

### 配置项位置

在 `config.ini` 文件的 `[model]` 部分：

```ini
[model]
local_model_path = D:\models\text2vec-base-chinese
vector_dimension = 768
default_provider = zhipu
vector_model_provider = local  ; 新增：向量模型提供商
```

### 可选值

- `local` - 使用本地模型
- `zhipu` - 使用智谱 AI

## 使用方法

### 方法 1：通过 Web 界面配置（推荐）

1. 打开系统首页
2. 点击"系统信息"标签
3. 在"配置信息"部分找到"向量模型提供商"
4. 选择"本地模型 (text2vec)"或"智谱 AI (embedding-2)"
5. 点击"保存配置"按钮
6. **重启服务器**使配置生效

### 方法 2：直接修改配置文件

1. 打开 `config.ini` 文件
2. 在 `[model]` 部分添加或修改：
   ```ini
   vector_model_provider = local
   ```
3. 保存文件
4. 重启服务器

## 配置生效

修改向量模型配置后，需要**重启服务器**才能生效。

重启后，系统会：
1. 根据配置加载相应的向量模型
2. 使用新模型进行文本向量化
3. 搜索结果会使用新模型生成的向量

## 注意事项

### 切换到本地模型

- 确保已下载 text2vec-base-chinese 模型
- 模型路径配置正确（默认：`D:\models\text2vec-base-chinese`）
- 首次加载可能需要较长时间

### 切换到智谱 AI

- 确保已配置智谱 AI API Key
- 确保网络连接正常
- 向量化速度取决于网络状况

### 已有数据

切换向量模型后：
- **已有文档的向量不会自动更新**
- 建议进行**全量重建**以使用新模型重新向量化所有文档
- 或者等待文档自然更新

## 技术实现

### 后端

1. **config.py** - 添加 `VECTOR_MODEL_PROVIDER` 配置项
2. **vector_store.py** - `EmbeddingModel` 和 `RedisVectorStore` 支持动态选择模型
3. **api_server.py** - 配置界面和 API 接口

### 前端

- 配置界面新增下拉框
- 保存配置时包含 `vector_model_provider` 字段

## 常见问题

### Q: 切换模型后搜索效果变差？

A: 可能需要重新向量化文档。建议：
1. 切换到新模型
2. 执行"全量重建"
3. 等待重建完成

### Q: 本地模型加载失败？

A: 检查以下几点：
1. 模型路径是否正确
2. 模型文件是否完整
3. 内存是否充足
4. 查看日志中的详细错误信息

### Q: 智谱 AI 向量化失败？

A: 检查：
1. API Key 是否配置
2. 网络连接是否正常
3. API 服务是否可用
