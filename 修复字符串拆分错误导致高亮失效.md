# 修复字符串拆分错误导致高亮失效的问题

## 问题描述

正则查询的"查看全部"功能中，高亮仍然没有显示。经过深入排查，发现问题出在字符串拆分上。

## 根本原因

在 `formatSmartContent` 和 `formatSectionContent` 函数中，使用了错误的字符串拆分方式：

```javascript
// ❌ 错误：使用双反斜杠，会按照字面意义的 "\n" 两个字符拆分
let lines = content.split('\\n');
let sections = content.split(/\\n\\s*\\n/);
```

这导致：
1. 内容无法正确按行拆分
2. 所有内容被当作一行处理
3. 高亮函数虽然被调用，但无法正确处理内容

## 问题分析

### JavaScript 字符串中的转义

在 JavaScript 中：
- `'\n'` 表示换行符（一个字符）
- `'\\n'` 表示反斜杠 + n 两个字符

当从服务器获取的文件内容时，内容中的换行符是真实的换行符 `\n`，而不是字符串 `\\n`。

### 正确的拆分方式

```javascript
// ✅ 正确：使用单反斜杠，按照真实的换行符拆分
let lines = content.split('\n');

// ✅ 正确：使用正则表达式，匹配换行符
let sections = content.split(/\n\s*\n/);
```

## 修复内容

### 1. formatSmartContent 函数

**修改前**：
```javascript
let lines = content.split('\\n');
```

**修改后**：
```javascript
let lines = content.split('\n');
```

### 2. formatSectionContent 函数

**修改前**：
```javascript
let sections = content.split(/\\n\\s*\\n/);
```

**修改后**：
```javascript
let sections = content.split(/\n\s*\n/);
```

## 高亮流程

### 正确的处理流程

```
1. 获取文件内容（包含真实的换行符 \n）
   ↓
2. 使用 content.split('\n') 按行拆分
   ↓
3. 逐行处理并应用高亮函数
   ↓
4. highlightRegexContent(text, pattern) 高亮匹配的内容
   ↓
5. 返回带 <span class="highlight"> 标签的 HTML
```

### 示例

假设文件内容：
```
用户手机号：13800138000
联系电话：15900159000
```

**修复前**：
```javascript
content.split('\\n')  // 返回：["用户手机号：13800138000\n联系电话：15900159000"]
// 只有一行，无法正确处理
```

**修复后**：
```javascript
content.split('\n')  // 返回：["用户手机号：13800138000", "联系电话：15900159000"]
// 正确按行拆分，每行都能高亮
```

## 经验总结

### JavaScript 转义字符

| 写法 | 含义 | 用途 |
|-----|------|------|
| `'\n'` | 换行符 | 拆分文本行 |
| `'\\n'` | 反斜杠 + n | 匹配字面意义的 "\n" 字符串 |
| `/\n/` | 匹配换行符的正则 | 正则表达式中匹配换行 |
| `/\\n/` | 匹配反斜杠 + n 的正则 | 匹配字面意义的 "\n" |

3. 检查每行内容是否正确
